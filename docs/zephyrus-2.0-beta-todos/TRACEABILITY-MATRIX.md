# Traceability Matrix

Purpose: assign stable requirement IDs to the Zephyrus 2.0 beta PRD so agents can implement, review, test, screenshot, deploy, and release against unambiguous acceptance rows.

Status values:

- `Open`: not implemented or not proven.
- `Partial`: code exists but proof, parity, security, or operations evidence is incomplete.
- `Ready for B8`: implemented and locally proven; final release evidence still needed.
- `Deferred`: not in beta; must appear in known limitations.
- `Complete`: implemented, reviewed, deployed or deployment-ready, and included in the final evidence package.

## Matrix Rules

- Every implementation PR must name the requirement IDs it changes.
- Every test, screenshot, API sample, and deployment smoke artifact must name the requirement IDs it proves.
- A requirement cannot move to `Complete` until its owning phase exit gate and B8 artifact expectations are satisfied.
- A requirement marked `Deferred` must also appear in `docs/beta-known-limitations.md` or the agreed limitations register.
- If a PRD requirement spans web, Eddy, and Hummingbird, all three surfaces must be represented unless the decision register explicitly scopes one out.
